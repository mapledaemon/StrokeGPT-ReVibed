===========================
StrokeGPT-ReVibed Changelog
===========================

This changelog tracks the merged pull requests made in the StrokeGPT-ReVibed
fork. It starts from the fork baseline and records the project work that has
landed through PRs in this repository.


2026-04-21 - PR #6 - Improve Motion Control and Local Runtime Setup
--------------------------------------------------------------------

Merged: https://github.com/mapledaemon/StrokeGPT-ReVibed/pull/6
Branch: codex-split-frontend-assets
Head commit: d0e8780
Merge commit: 3d3b7f8

- Split the large inline browser UI into `static/app.css` and `static/app.js`
  while keeping `index.html` as the page markup.
- Added reusable Handy motion pattern definitions and expanded spatial motion
  mapping so LLM output can better express tip, base, full-length, stroke
  length, and pattern cues.
- Improved smooth motion transitions and pattern planning while preserving user
  speed limits and the explicit stop path.
- Added Chatterbox Turbo defaults, local Torch/CUDA diagnostics, chunked local
  voice generation, engine/style controls, and clearer CPU-only latency
  warnings.
- Made large model downloads explicit in the UI: Ollama downloads now live in
  the Model tab, and local Chatterbox model download/load lives in the Voice
  tab.
- Stopped automatic local voice model download/preload during startup and
  settings saves.
- Added local port fallback so an occupied default Flask port no longer blocks
  startup.
- Moved regression tests into the `tests/` folder and updated local and CI test
  commands.
- Rewrote install instructions with separate Windows, macOS, and Linux flows.

2026-04-21 - PR #4 - Organize Portable Settings
------------------------------------------------

Merged: https://github.com/mapledaemon/StrokeGPT-ReVibed/pull/4
Branch: codex-settings-portability-reset
Head commit: 445bbe7
Merge commit: 4ae1c9c

- Reorganized the settings dialog into Persona, Model, Voice, Device, Motion,
  and Advanced tabs.
- Moved the Handy connection key and stroke range controls into the Device tab.
- Added speed limit controls to the Motion tab so limits from initial setup can
  be adjusted later.
- Added saved persona prompt presets, including girlfriend, boyfriend, and
  partner defaults.
- Centralized settings defaults, serialization, migration, and reset behavior
  around a portable settings schema.
- Added a confirmed Reset All Settings action that stops active motion, resets
  runtime state, clears saved settings, and returns the app to setup.
- Fixed local Chatterbox warning suppression so loading no longer fails with
  unsupported `catch_warnings` arguments.
- Updated README and Codex handoff notes for the new settings layout.
- Added regression coverage for the new settings UI and backend routes.
