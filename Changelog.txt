===========================
StrokeGPT-ReVibed Changelog
===========================

This changelog tracks pull requests made in the StrokeGPT-ReVibed fork, plus
the current unreleased branch only before a pull request exists. Once a branch
has an open PR, its changelog entry should use the PR number instead of
`Unreleased` so follow-up changelog-only PRs are not needed.


2026-04-23 - PR #48 - Extract Freestyle Planner And Mode Decision Helpers
-------------------------------------------------------------------------

Branch: codex-background-modes-split

- Split `strokegpt/background_modes.py` into three focused modules to make
  the mode-orchestration layer easier to read and extend without touching
  the scoring or decision-parsing code paths. Behavior is preserved; this
  is a mechanical extraction with re-exports so existing imports and tests
  keep working.
- Added `strokegpt/mode_decisions.py` owning `ModeDecision`,
  `MODE_DECISION_ACTIONS`, `_coerce_mode_decision`,
  `_request_mode_decision`, `_start_mode_decision_request`,
  `_poll_mode_decision_request`, `_step_limit_for_duration`,
  `_target_with_intensity`, and `_send_mode_decision_message` so the LLM
  mode-decision contract lives in one testable place.
- Added `strokegpt/freestyle.py` owning the Freestyle planner helpers:
  `FreestyleChoice`, the `FREESTYLE_*` constants, candidate shaping
  (`_candidate_*`), profile/blend/score/target helpers, edge-reaction
  planning (`_edge_reaction_steps`, `_apply_freestyle_edge_reaction`),
  chain planning (`_freestyle_choice_chain`, `_freestyle_choice_frames`,
  `_apply_freestyle_choices`, `_record_freestyle_edge_playback`), and the
  permission/close-style helpers used by the Freestyle loop.
- Reduced `strokegpt/background_modes.py` to orchestration only:
  `AutoModeThread`, `_check_for_user_message`, `_feedback_target`,
  `_wait_while_paused`, `_sleep_with_stop`, `_set_active_mode`,
  `_run_scripted_mode`, and the public `auto_mode_logic`,
  `freestyle_mode_logic`, `milking_mode_logic`, `edging_mode_logic` entry
  points. Module dropped from ~1000 lines to ~593; the new
  `mode_decisions.py` (~154 lines) and `freestyle.py` (~356 lines) hold
  the extracted logic.
- Re-exported the split symbols through `background_modes` so existing
  imports in `web.py` and the `background_modes._coerce_mode_decision`,
  `background_modes._choose_freestyle_pattern`, and
  `background_modes.random` references used by the test suite continue
  to resolve to the correct callables without touching the tests.
- Verified with `python -m unittest discover -s tests` (224 tests pass,
  same 69 Flask-dependent tests skip as before) and
  `python -m py_compile` across `background_modes.py`, `freestyle.py`,
  `mode_decisions.py`, and `web.py`.

2026-04-23 - PR #43 - Improve Freestyle Diagnostics And Motion Hot Paths
-------------------------------------------------------------------------

Branch: codex-freestyle-diagnostics-caching

- Added a mode-agnostic elapsed timer to `/get_status` so Auto, Edge, Milk,
  and Freestyle can show active runtime without depending on the edging prompt
  context.
- Detached the motion sequence indicator from the speed/depth meter and changed
  it into a vertical, terminal-style recent-sequence log with duplicate
  suppression.
- Moved the motion sequence log to the left of the speed/depth indicator bar,
  reset it when a new active-mode timer starts, and changed log timecodes to
  use the active-mode elapsed timer instead of wall-clock time.
- Widened the terminal-style motion log and tightened the paired indicator bar
  layout so the log, speed/depth meters, and feedback buttons align without
  loose spacing.
- Fixed the motion status strip height so terminal-log content cannot resize
  the footer section.
- Let the motion sequence log include recent trace labels, speed/depth/range,
  backend state, frame position, command timing, and gap timing as diagnostics
  verbosity increases.
- Made the motion diagnostics panel visible at status/debug levels and surfaced
  PR #42 trace fields so Freestyle testing can distinguish planner gaps,
  command latency, pass-through final frames, and backend/device state.
- Restored the active-mode elapsed display to clock formatting and guarded Edge
  start decisions so a short LLM start duration or start `stop` action cannot
  immediately end the mode.
- Rewrote the LLM mode-decision prompt to be shorter and more direct about
  avoiding very short durations, with a 10-second minimum duration clamp for
  parsed mode decisions.
- Tightened chat, repair, naming, GLaDOS, and memory-consolidation prompts
  while preserving direct erotic language and motion-control constraints.
- Biased chat and mode-decision prompts so mode starts usually begin
  base-through-mid or mid-base, with tip/full-travel extension later unless the
  user explicitly asks for tip focus.
- Rebalanced Edge and Milk scripted arcs so their early mode steps favor
  base-to-mid targets before introducing tip-focused motion.
- Changed Edge's random session cap into a progress decision checkpoint so the
  LLM can continue, hold, pull back, switch to Milk, or deliberately stop
  instead of letting Edge fall through to an abrupt automatic stop.
- Pre-compiled the motion intent and cue regex tables (ZONE/LENGTH/PATTERN/
  SPEED) and IntentMatcher's STOP/CONTROL/INFORMATIONAL groups at import time
  so chat parsing and frame-time intent matching no longer recompile patterns.
- Hoisted IntentMatcher's per-call faster/slower/harder/gentler/deeper/
  shallower/anchor-program literal tuples to module-level compiled-pattern
  constants so each invocation reuses the same Pattern objects.
- Pre-compiled the slugify and whitespace-normalize regexes used by motion
  cue detection and pattern-id slugging.
- Added an lru_cache around the controller's label-to-pattern lookup so
  repeated chat/LLM labels resolve from cache instead of re-sorting and
  re-scanning the static PATTERNS dict on every generated target.
- Added an lru_cache around prepare_pattern_actions in motion_patterns.py
  so each pattern's normalize/repeat/interpolate/simplify pipeline runs once
  per pattern instead of once per playback expansion and once per duration_ms
  lookup.
- Removed the duplicate _slug_pattern_id helper in background_modes.py and
  re-exported the canonical motion._slugify_motion_pattern_id under the same
  local name so the freestyle planner and the controller share one
  implementation.


2026-04-22 - PR #42 - Bridge Position Playback And Trace Timing
----------------------------------------------------------------

Branch: codex-freestyle-bridge-and-trace-timing

- Added a controller-level current-position bridge to the flexible position
  backend so the existing depth-step splitter also smooths the jump from the
  device's last commanded depth into the first frame of every batch, mirroring
  the HAMP transition path used by the default backend.
- Hardened the LLM mode-decision coercion so a `stop` action emitted on the
  start of a continuous mode (Milking or Freestyle) is downgraded to
  `continue`, matching the prompt contract and preventing local models from
  ending continuous modes by their own start decision.
- Added per-frame timing fields (command_ms, gap_ms, batch_gap_ms,
  frame_index, frame_count, is_pass_through_final) to the position-backend
  observability trace so the next on-device test can pinpoint whether a
  Freestyle stop is planner-side or API-side without changing playback.
- Updated motion-control tests to cover the new leading bridge frame and the
  new trace timing fields, and added a coercion test for the continuous-mode
  start guard.


2026-04-22 - PR #41 - Add Adaptive Freestyle Planner
----------------------------------------------------

Branch: codex-adaptive-freestyle-planner

- Wired the Experimental Freestyle control to start an adaptive planner instead
  of leaving it disabled.
- Added fuzzy pattern selection over enabled motion patterns using visible
  weights, feedback, recent pattern history, current motion context, and chat
  feedback relayed during the mode.
- Kept Freestyle off the HAMP scripted Auto arcs by playing selected pattern
  frames through the experimental position/script playback path.
- Chained Freestyle pattern playback with pass-through final targets so the
  device slides from one selected pattern into the next instead of stopping at
  each pattern boundary.
- Queued short Freestyle pattern chains into each playback call and removed the
  nonzero boundary pause so adaptive mode does not stop at regular selection
  intervals.
- Removed the zero-duration Freestyle sleep floor that was still creating a
  regular boundary pause between adaptive chains.
- Made the I'm Close control visible during Freestyle and routed it to the LLM
  mode-decision prompt so Freestyle can bias toward edge-style or milk-style
  motion instead of stopping automatically.
- Changed Freestyle edge-style close decisions to run the same pullback,
  recover, and hold reaction sequence used by Edge Me before returning to
  normal Freestyle selection.
- Added a safe edge-motion buffer while Freestyle waits for the LLM to decide
  after I'm Close, so local model latency does not create a motion gap.
- Made Milk Me continuous until the user stops it or explicitly changes mode,
  instead of ending after the LLM's initial duration hint or a fixed step cap.
- Made Edge-to-Milk transitions enter continuous Milk Me rather than a finite
  milk segment that can stop shortly after switching modes.
- Updated mode-decision prompting so Milking and Freestyle are described as
  continuous modes, with duration treated as a temporary timing hint rather
  than a completion instruction.
- Added Motion settings permissions for allowing LLM edge behavior in Freestyle
  close decisions and in normal chat-generated motion.
- Filtered chat-visible edge patterns out of the LLM motion prompt when chat
  edge permission is disabled, while keeping deterministic Edge Me available.
- Added shared blend frames to pattern and anchor expansion so new scripts ease
  out of the previous motion state before entering the next pattern shape.
- Added lookahead turn-apex smoothing for intra-pattern direction changes so
  the controller brakes into reversal points and eases out of them instead of
  snapping directly into the next stroke direction.
- Added position-playback guard rails that split oversized depth jumps into
  smaller XAVA targets and cap generated velocities against the current Handy
  max-speed setting.
- Slowed final pass-through XAVA targets so Freestyle batch handoffs have time
  to retarget before the device reaches the end of a finite command sequence.
- Routed plain chat-generated targets through the flexible position backend
  when selected, so numeric chat control receives the same smoothing and
  velocity guard rails as pattern/script playback.
- Updated the roadmap so raw on-the-fly motion generation remains a later
  guarded target behind graph preview, validation, smoothing, and safety
  constraints.
